# 26 — CRM (Customer Relationship Management)

> Создано 2026-05-04. Первая версия CRM v0.1 поднята вместе с backend.

## Что это

CRM в V7 — это набор инструментов чтобы **знать каждого клиента** и **действовать** на основе этого знания:
- Кто он (контакты, уровень, предпочтения)
- Что играл (история бронирований, паттерны)
- Что покупал (выручка, средний чек, категории)
- Что должны помнить о нём (заметки админов, теги, конфликты)
- Когда последний раз контактировали и что было результатом

## Что НЕ есть V7 CRM

- ❌ **Не PCRM** (личный networking-CRM Володимира) — другая система с другими данными
- ❌ **Не Matchpoint** (это booking-engine, не CRM)
- ❌ **Не HubSpot/Salesforce** (overengineered для нас, vendor-lock)

## Use cases

### 1. Ресепшн при бронировании
**Кто**: админ ресепшн
**Когда**: каждый день, 5 секунд на клиента
**Что нужно**: имя, контакты, статус (member/нет), баланс, есть ли долги, конфликты
**Where**: модалка из /clients (быстрый view)

### 2. Manager при retention call
**Кто**: Настя/Оля
**Когда**: weekly (cold/slow клиенты из Top-60)
**Что нужно**: всё из выше + история игр + последние заметки + что покупал + когда последний раз контактировали
**Where**: модалка drill-down (расширенная)

### 3. Володимир при анализе
**Кто**: Володимир
**Когда**: weekly review, перед турнирами, при стратегических решениях
**Что нужно**: глубокая карточка с timeline, паттернами, всеми тегами/заметками
**Where**: отдельная страница /client.html (план)

### 4. Captain Inner Circle / тренер
**Кто**: тренер/captain
**Когда**: после игры — записать заметку «играл с N, понравилось»
**Что нужно**: быстрое добавление заметки + tag
**Where**: квики из любой страницы (план)

## Текущая реализация (v0.1)

### Что сделано
- Backend с notes/tags/contact_log endpoints
- Auth через тот же password что login.html (JWT 24h)
- UI блок CRM в модалке клиента на /clients:
  - Список тегов + добавить/удалить
  - Список заметок + добавить/удалить
  - Кнопка «Записать контакт» (channel/result/note)

### Что НЕ работает (в backlog)
- ⏳ Отдельная full-page карточка клиента (`/client.html?cid=N`)
- ⏳ AI-генерируемые заметки на основе истории игр
- ⏳ Bulk operations (отметить группу как «позвонили»)
- ⏳ Calendar integration (запись «следующий контакт через 7 дней»)
- ⏳ WhatsApp action button (с pre-filled template) — после Meta approval
- ⏳ Conflict graph («не пересекать с X»)
- ⏳ Timeline view (все события в хронологии)
- ⏳ Inner Circle tag + automation (notify в чат)
- ⏳ Photos / portrait в карточке (для memory recall)

## Принципы

### 1. Action-oriented, не information-overload
Карточка должна провоцировать **действие**, не только показывать данные. Каждый блок должен иметь next-step (позвонить / отправить / записать).

### 2. Skin in the game — кто пишет, тот и подписан
Notes/contact_log имеют `author` поле (email/login). Нельзя писать анонимно. Это формирует культуру ответственности.

### 3. Append-only, не редактирование
Заметки нельзя редактировать после создания (можно удалить + написать новую). Это сохраняет timeline честным — без переписывания истории.

### 4. Privacy by design
PII никогда не попадает в backend SQLite. Только cid + текст. Если backend будет compromised — утечёт что 38=«VIP, любит утра», но без понимания кто такой 38 (это требует Sheets с PII).

### 5. Multi-tenant ready
Все CRM-таблицы имеют `club_id`. Когда откроется 2-й клуб — фильтр в API, без миграции.

## Метрики успеха CRM

(Когда наберём данных — отдельный дашборд)

- Сколько % клиентов имеют ≥1 заметку
- Сколько % имеют ≥1 контакт за 30 дней
- Average notes per active client
- % контактов с положительным результатом
- Adoption: сколько админов активно пишут заметки (per-author count)

## Грабли (учиться на ошибках)

- 🚫 НЕ дублируем заметки в Google Sheets (concurrency проблемы)
- 🚫 НЕ пишем PII в notes («вчера звонила Васе, Аня плакала» → достаточно «вчера созвон, расстроена», без Васи и Ани)
- 🚫 НЕ ставим теги без agreed taxonomy (lead → Володимир): «VIP», «инфлюенсер», «любит коуча X», «family», «turn-friendly» — фиксированный список
