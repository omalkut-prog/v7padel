# 26 — CRM (Customer Relationship Management)

> Создано 2026-05-04. Первая версия CRM v0.1 поднята вместе с backend.

## Что это

CRM в V7 — это набор инструментов чтобы **знать каждого клиента** и **действовать** на основе этого знания:
- Кто он (контакты, уровень, предпочтения)
- Что играл (история бронирований, паттерны)
- Что покупал (выручка, средний чек, категории)
- Что должны помнить о нём (заметки админов, теги, конфликты)
- Когда последний раз контактировали и что было результатом

## Что НЕ есть V7 CRM

- ❌ **Не PCRM** (личный networking-CRM Володимира) — другая система с другими данными
- ❌ **Не Matchpoint** (это booking-engine, не CRM)
- ❌ **Не HubSpot/Salesforce** (overengineered для нас, vendor-lock)

## Use cases

### 1. Ресепшн при бронировании
**Кто**: админ ресепшн
**Когда**: каждый день, 5 секунд на клиента
**Что нужно**: имя, контакты, статус (member/нет), баланс, есть ли долги, конфликты
**Where**: модалка из /clients (быстрый view)

### 2. Manager при retention call
**Кто**: Настя/Оля
**Когда**: weekly (cold/slow клиенты из Top-60)
**Что нужно**: всё из выше + история игр + последние заметки + что покупал + когда последний раз контактировали
**Where**: модалка drill-down (расширенная)

### 3. Володимир при анализе
**Кто**: Володимир
**Когда**: weekly review, перед турнирами, при стратегических решениях
**Что нужно**: глубокая карточка с timeline, паттернами, всеми тегами/заметками
**Where**: отдельная страница /client.html (план)

### 4. Captain Inner Circle / тренер
**Кто**: тренер/captain
**Когда**: после игры — записать заметку «играл с N, понравилось»
**Что нужно**: быстрое добавление заметки + tag
**Where**: квики из любой страницы (план)

## Текущая реализация (v0.1)

### Что сделано
- Backend с notes/tags/contact_log endpoints
- Auth через тот же password что login.html (JWT 24h)
- UI блок CRM в модалке клиента на /clients:
  - Список тегов + добавить/удалить
  - Список заметок + добавить/удалить
  - Кнопка «Записать контакт» (channel/result/note)

### Что НЕ работает (в backlog)
- ⏳ Отдельная full-page карточка клиента (`/client.html?cid=N`)
- ⏳ AI-генерируемые заметки на основе истории игр
- ⏳ Bulk operations (отметить группу как «позвонили»)
- ⏳ Calendar integration (запись «следующий контакт через 7 дней»)
- ⏳ WhatsApp action button (с pre-filled template) — после Meta approval
- ⏳ Conflict graph («не пересекать с X»)
- ⏳ Timeline view (все события в хронологии)
- ⏳ Inner Circle tag + automation (notify в чат)
- ⏳ Photos / portrait в карточке (для memory recall)

## Принципы

### 1. Action-oriented, не information-overload
Карточка должна провоцировать **действие**, не только показывать данные. Каждый блок должен иметь next-step (позвонить / отправить / записать).

### 2. Skin in the game — кто пишет, тот и подписан
Notes/contact_log имеют `author` поле (email/login). Нельзя писать анонимно. Это формирует культуру ответственности.

### 3. Append-only, не редактирование
Заметки нельзя редактировать после создания (можно удалить + написать новую). Это сохраняет timeline честным — без переписывания истории.

### 4. Privacy by design
PII никогда не попадает в backend SQLite. Только cid + текст. Если backend будет compromised — утечёт что 38=«VIP, любит утра», но без понимания кто такой 38 (это требует Sheets с PII).

### 5. Multi-tenant ready
Все CRM-таблицы имеют `club_id`. Когда откроется 2-й клуб — фильтр в API, без миграции.

## AI Agent Roadmap (одобрено Володимиром 2026-05-04)

### Главная идея

V7 строит **инфраструктуру коммуникации** которая позволит покупать другие клубы и делать их эффективными по AI. То что сейчас работает (community, ETL, dashboards) — продолжает работать. AI-агент **дополняет**, а не заменяет.

**Сравнение с индустрией**: ни один padel-клуб в Турции (и мало кто в мире) не имеет CRM где AI-агент знает каждого клиента лично. Большинство = WhatsApp в общем чате 200 человек. **Это разрыв на 2-3 года вперёд**.

### Каналы которые учитываем

V7 общается через 4 канала + 1 встроенный:
1. **WhatsApp Business** (главный для турок и экспатов)
2. **Telegram** (RU/UA сообщество)
3. **Instagram DM** (туристы, новички)
4. **Matchpoint mailings** (членам клуба, по системе)
5. **Внутренний портал V7** (admin notes, что говорить)

Agent должен работать со всеми каналами параллельно — клиент может писать через любой, agent отвечает в том же.

### Приоритетный язык — multilingual

В Анталии: турки → экспаты (RU/UA/EU/Asia) → туристы. Agent отвечает **на языке клиента**: турецкий, английский, русский, украинский, испанский (для падел-сообщества).

Это конкретное конкурентное преимущество — никто не делает.

### KVKK / Privacy — opt-in обязательно

При registration / membership signup — **галочка согласия на профайлинг**:
> «Мы собираем данные о ваших играх и предпочтениях чтобы предлагать релевантные ивенты. Вы можете отозвать согласие в любой момент.»

Без этой галочки — клиент **не попадает** в outreach pipeline. Только в admin tools (агент помогает админу, но не пишет клиенту).

### Phased approach

Phase 1 → 2 → 3 → 4 (см. ниже). НЕ скакать через фазы.

#### Phase 1 — Admin Agent (2-3 недели)
**Кому пишет**: только admin/Володимиру, клиент НЕ видит agent.

**Use cases**:
- «Найди игроков на американо в пятницу 20:00 для новичков» → список + drafts
- «ДР в ближайшие 7 дней» → список + suggestion как поздравить
- «VIP/Club — кому давно не делали touch» → cold list
- «Кто перестал ходить» → recall candidates
- «Аналитика VIP/Club: уровень, пол, любимое время, ДР»

**Tech**: backend `/api/agent/query` + Claude API + structured prompts. Cost ~$10/мес.

**Risk**: минимальный — клиент не получает прямые сообщения от AI.

#### Phase 2 — Survey Agent (1-2 недели после Phase 1)
**Задача**: заполнить пробелы в данных через приятный onboarding опрос. Pre-condition для Phase 3-4.

**Что собирает**: уровень, язык общения, любимое время, любимый тип игры, interests, предпочтительный канал коммуникации.

**Через**: WhatsApp template (approved Meta) или встроенный web-опросник.

#### Phase 3 — Inbox-aware Agent (после Phase 2)
**Задача**: клиент **пишет V7** через любой канал — agent отвечает в его контексте.

**Принцип**: agent НЕ инициирует, только отвечает. KVKK безопасно (клиент сам начал диалог).

**Пример**:
> Клиент (WhatsApp): «есть слот на 4 в субботу вечером?»
> Agent: «Привет Андрей, вечером субботы свободно P2 18:00 и P3 20:00 — какой удобнее?»
> (зная: cid=42, Intermediate, обычно играет P2 в 18:00)

#### Phase 4 — Outreach Agent (через 3-6 месяцев)
**Задача**: agent сам инициирует — приглашения на ивенты, ДР, recall, события.

**Только после**: Phase 1-3 показали traction + WhatsApp templates approved + opt-in покрывает большую часть базы.

### План тестирования (одобрено Володимиром)

> «Я строю инфраструктуру. То что работает, пусть работает. Просто с таким подходом я смогу покупать клубы и делать их эффективными по AI.»

**Этапы запуска outreach (Phase 4)**:

1. **Сначала Володимир сам** — agent пишет ему. Володимир =~ пользователь, видит баги, корректирует.
2. **Потом 5 близких людей / партнёров** — Артём, Оля, тренеры, кто простит ошибки.
3. **Потом 10-15 лояльных клиентов-друзей** — те кто играет давно, простят промах, дадут feedback.
4. **Замеряем всё** — open rate, response rate, conversion, complaints, unsubscribes.
5. **Улучшаем настройки** на основе данных.
6. **Только потом 50-200 клиентов** — массовый запуск с полной уверенностью.
7. **Полная база (1500+)** — после 50-200 показали стабильные метрики.

**Принцип**: НИКОГДА не запускаем agent на массовую аудиторию без тестирования на узком кругу. Нарушение этого принципа = потеря доверия (как блокировка WhatsApp за злоупотребление).

### Lessons learned (грабли которых избегаем)

- 🚫 **WhatsApp заблокировали** в прошлом потому что использовали без templates → массовые сообщения trigger spam-detection. **Никогда** не повторять — только approved templates.
- 🚫 **Переоценивать готовность** клиентов к AI-общению. Многие пугаются «а откуда вы знаете мой ДР» → нужен contextual onboarding.
- 🚫 **Generic шаблоны** «играет по утрам, любит P2» — нужны как **starter context**, но agent должен дополнять real insights из истории.
- 🚫 **Пропускать opt-in** — KVKK штраф до 10М₺ за нарушение.

### Главная цель агента

> «Я хочу чтоб агент обращался по имени, знал что любит игрок, на нужном языке, предлагал ему только подходящее. Если не знает что подходит — провёл бы анкетирование в максимально удобной форме.»
>
> — Володимир, 2026-05-04

Это **закрытый цикл**: знаем → предлагаем → играют → знаем больше. Self-reinforcing.

## Метрики успеха CRM

(Когда наберём данных — отдельный дашборд)

- Сколько % клиентов имеют ≥1 заметку
- Сколько % имеют ≥1 контакт за 30 дней
- Average notes per active client
- % контактов с положительным результатом
- Adoption: сколько админов активно пишут заметки (per-author count)

## Грабли (учиться на ошибках)

- 🚫 НЕ дублируем заметки в Google Sheets (concurrency проблемы)
- 🚫 НЕ пишем PII в notes («вчера звонила Васе, Аня плакала» → достаточно «вчера созвон, расстроена», без Васи и Ани)
- 🚫 НЕ ставим теги без agreed taxonomy (lead → Володимир): «VIP», «инфлюенсер», «любит коуча X», «family», «turn-friendly» — фиксированный список
