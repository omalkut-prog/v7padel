# 06 — Principles

Как мы работаем. Стабильные правила, пересматриваются раз в квартал, не чаще.

## Стратегические (бизнес)

1. **Премиум-тир — это ДНК**. Качество > дешевизна. Цена — часть продукта.
2. **Никаких скидок**. Объёмные пакеты и турнирные воучеры — это не скидки, а часть продукта.
3. **Один North Star** — 80 k₺/день (30-дневная скользящая). Всё приоритизируется через влияние на него.
4. **4 контура роста** — аренда, абонементы, тренировки, бар/магазин. Балансируем, не перекашиваем.
5. **Цели в числах с дедлайном**. «Улучшить» — не цель. «С X до Y к дате» — цель.

## Операционные (работа с продуктом)

6. **Данные важнее фич**. Сначала чистые данные, потом красивый UI поверх.
7. **Фиксить причину, а не симптом**. Увидел мусорные строки — фикси парсер, а не фильтры сверху.
8. **Малые циклы**. Dry-run → audit → write. Никогда не пишем в прод без dry-run.
9. **Один дизайн-система**. Никаких hardcoded :root theme на отдельных страницах.
10. **Dry-run по умолчанию**. ETL и любые записи в Sheets — без `--write` ничего не пишем.

## Дисциплина документации

11. **Doc-in-commit**. Меняешь код → обновляешь doc в том же PR. Либо флаг `NO_DOC: причина` в коммит-сообщении.
12. **Numbers только из `04_CURRENT_STATE.md`**. Не копировать цифры в другие доки.
13. **ADR на каждое неочевидное решение**. 5 строк в `09_DECISIONS.md`: контекст, варианты, выбор, почему, дата.
14. **Anti-pattern после каждого факапа**. 1 строка в `07_ANTI_PATTERNS.md` с ссылкой на коммит.
15. **Max 5 в `NOW`**. Force priority discipline.

## Коммуникация

16. **Русский язык**. Все ответы и документы на русском. Английские вставки только для технических терминов без русских аналогов (API, ViewState, ETL).
17. **Не дублировать данные**. Если число уже есть в другой вкладке/файле — ссылка, не копипаст.
18. **Приоритет + тип + длительность + исполнитель**. Четыре измерения задачи, не смешивать.
19. **Думать → предлагать варианты → рекомендовать**. Не «первое попавшееся», а обдуманное с pros/cons.
20. **Сразу продуманные предложения**. Если есть выбор, показать 2-3 варианта с трейдоффами, не первую идею.

## Исключения и граничные случаи

**Когда эти принципы не применяются**:
- Hot-fix критического бага в проде может идти без doc-update (но сразу после — обновить и добавить в `07_ANTI_PATTERNS`)
- Чисто экспериментальный probe-скрипт не обязан следовать dry-run паттерну
- ADR не пишется для очевидных рутинных решений

Это **исключения**, не лазейки. Сомневаешься — пиши doc.
