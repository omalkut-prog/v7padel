## 🏛 Архитектура

**Источники данных:**
- **Matchpoint** — операционная CRM: bookings, customers, memberships. ASP.NET WebForms, двухшаговый логин (credentials → cashbox). ✅ **АВТОМАТИЗИРОВАН** — еженедельный синк customers через `sync_customers_matchpoint.py`.

- **Racket.ID** — ✅ **ПОДКЛЮЧЕН** через Firestore REST API (публичный доступ без авторизации). 351 участник, 338 турниров, 2,597 записей участия.

## 📜 Принципы

1. **Не автоматизируй плохой процесс — убери его.**
2. **Один источник правды.** PnL = бухгалтер. Клиенты = Matchpoint. Турниры = Racket.ID.
3. **Всё измеряется.** Если метрику нельзя посчитать — фича не считается готовой.
4. **Данные вводятся один раз.** Повторный ввод = источник ошибок.
5. **North Star фильтр.** Двигает ли это 80k ₺/день?
6. **Масштабируемость.** Работает в сети из 10 клубов, а не только в Анталье.
7. **Простота важнее полноты.** Лучше 5 рабочих цифр, чем 50 которые никто не понимает.
