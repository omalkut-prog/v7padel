V7 Padel Brain

10 разделов правил + спецификация v7padel_db · обновлено 2026-04-13

1 · Языки

  • Сайт (CEO/Оля): русский — основной
  • Персонал (турки): турецкий + английский
  • Турецкий — строгая орфография: İ Ş Ğ Ü Ö Ç обязательно
  • Если не уверен в турецком — пометка нужна проверка носителем
  • Никакого транслита

2 · Данные и источники

  • P&L — только из PnL бухгалтера (новый файл). Старый pnl_monthly в v7padel_db — deprecated
  • Названия колонок — точные, не угадывать
  • Fallback без пометки в UI — запрещён
  • Нет данных для блока = сообщи что добавить в ETL, не строй на заглушках
  • Нет данных в UI = явное сообщение с причиной, никогда пустой блок или вечный спиннер

Агрегация P&L (НОВЫЙ формат — PnL бухгалтера)

Источник: 1wLvhEUAsS08K6LvxSKAx5-soh5NxayMpU3brvd4rQ54, GID 1844112584

  • Месяцы — колонки (не строки). Каждый месяц = 4 колонки: Cashless, Cash, итого, Уд.вес
  • Revenue = строка «Доход от деятельности» → колонка «итого»
  • OPEX = строка «Условно-постоянные расходы» → |итого| (abs)
  • PnL = строка «Прибыль» → итого
  • Детализация: Падел, Bookings, Membership, Classes, Tournaments, Goods (revenue); Admin, Bank, Utilities, Marketing, Staff, Taxes (opex)

Fuzzy matching: bookings ↔ customers

  • Скрипт: scripts/fuzzy_match.py, результаты: data/booking_matches.csv
  • Покрытие: 1,027 бронирований (14.0%) привязаны к 267 уникальным клиентам
  • Методы: exact name (1,003 high), substring (24 medium)
  • Staff exclusion: Gulcan Yanar (бывший тренер, 255 бронирований = work records)
  • Не матчатся: турниры (team VS team), generic bookings, match records — нужен Racket.ID

4.5 · Matchpoint API

ASP.NET WebForms. API через PageMethods (JSON POST). Все вызовы требуют аутентифицированную сессию.

Base URL: https://app-v7padel-tur.matchpoint.com.es

Аутентификация (двухшаговая)

  1. GET /Login.aspx → извлечь __VIEWSTATE, __VIEWSTATEGENERATOR, __EVENTVALIDATION
  2. POST форму: username, password, ddlLenguaje=en-GB, __EVENTTARGET=btnLogin
  3. Если ответ содержит btnAcceder → POST выбор кассы: DropDownListCajas=1, __EVENTTARGET=btnAcceder

Креды: etl/matchpoint_creds.json. Реализация: etl/sync_customers_matchpoint.py → mp_login()

9 · Работа с ИИ (Claude Code)

Проект разрабатывается с помощью Claude Code (CLI). Для продолжения контекста ИИ использует файлы CLAUDE.md и V7_Padel_Brain.md.

Как запустить Claude Code

  1. Терминал (PowerShell или cmd): cd C:\Users\volod\v7padel\site
  2. Запуск: claude (или claude --dangerously-skip-permissions для автоматического режима)
  3. Описать задачу на русском. Давать конкретные числа для проверки.

Правила взаимодействия

9.1
Все ответы на русском. Без английских вставок в тексте.
9.2
Одна задача за раз. Не смешивать 1-минутную правку с 10-часовой фичей.
9.3
Сначала доделай, потом переходи к следующему.
9.4
Не дублируй данные. Если данные уже есть на другой вкладке — сообщи.
9.5
Метрики: выручка/корто-час важнее tx/день. Цели = скользящий тренд +15%.
9.6
Приоритет задач: приоритет + тип + длительность + исполнитель.
